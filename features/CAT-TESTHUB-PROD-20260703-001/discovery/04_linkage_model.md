## 2. Confirmed defects in current linkage code

- **D1 — LogDefectModal is a mock.** src/components/releases/test-execution/LogDefectModal.tsx:52-54 fabricates `DEF-###` with `Math.random()` and shows a success toast; **no DB write, no link**. Also violates color law (`text-red-600` etc., lines 63,121) and hand-rolled dialog.
- **D2 — CycleDetailPage DefectPanel writes garbage FK.** CycleDetailPage.tsx:709: `project_id: item.id // scope as project proxy` — inserts the *cycle-scope row id* into `tm_defects.project_id` (FK→tm_projects), and omits required `defect_key` (Insert type requires it, types.ts:43060). Insert either errors or corrupts project scoping.
- **D3 — useCreateDefect auto-link chain targets 3 relations missing from types** (`tm_defect_links` extended columns, `tm_requirement_tests`/`tm_requirements`, `plan_test_cycles`) — useDefects.ts:353-469. Either stale types or silently failing writes; matches memory lesson "cyij migration marked-not-executed / silent {data} destructure".
- **D4 — Story↔test split-brain** (`linked_story_key` vs `tm_requirement_links.external_key`) — two link models, two UIs, no sync (TestCasesSection vs TestCoveragePanel).
- **D5 — Traceability page fetches ALL orgs' links then filters client-side** (TraceabilityPage.tsx:62-79: no `.eq(project)` on the query; filter after join) and uses `projects[0]?.id` ignoring the route's projectKey (TraceabilityPage.tsx:119-121).
- **D6 — Color-law violations inside linkage UIs**: useRequirementLinks.ts:214-219 (`bg-yellow-100`, `text-green-700`, …), TestCaseLinksAttachments.tsx:45-58 (Tailwind color map), LogDefectModal (above). Pre-existing debt under ratchet, but these files will be touched by the revamp → must be fixed in-slice.
- **D7 — Dual defect universes.** QA Bug create writes `tm_defects` (CreateStoryModal.tsx:836-872) but CatalystViewDefect renders a ph_issues-shaped `issue.issue_key` (CatalystViewDefect.tsx:159) — the defect detail view and defect storage do not share a single spine; TestCoveragePanel(mode="defect") joins by key text.
