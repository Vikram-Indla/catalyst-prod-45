Root Cause & Fix

The editor was loading slowly because of two layers of lazy-loading:

❌
Layer 1 (Outer): CatalystDescriptionSection.tsx was using lazy(() => import(...)) to load EpicDescriptionEditor. This was converted to a static import.
❌
Layer 2 (Inner — ROOT CAUSE): Inside EpicDescriptionEditor.tsx, lines 31–36 were using lazy(async () => import('@atlaskit/editor-core')) to load the Editor component. This caused a 2–3 second delay and the "Loading editor..." spinner on every edit click.

The fix: Converted to static import: import { Editor } from '@atlaskit/editor-core' (line 27). The lazy-load was originally added (2026-04-20) to prevent ProseMirror double-registration collision with TipTap. TipTap was removed on 2026-04-20, so the lazy-load is no longer needed.

Known Issue — Not in This Fix: Image attachment rendering. When users paste or upload images, they go to Supabase (`supabaseImageUpload`), but the attachment metadata is queried by MediaProvidersShell from the ph_issue_attachments table — which lives in Lovable's database, not Supabase. This architectural split causes images to render with "hosted on Jira · auth required" status. Requires syncing attachment metadata between Lovable and Supabase, or refactoring the query path. Documented in `/catalyst/02_knowledge/lovable-database-ownership-supabase-architecture.md`.
