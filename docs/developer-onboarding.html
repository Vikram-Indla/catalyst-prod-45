Quick start (new dev)

1
Clone & install
git clone https://github.com/Vikram-Indla/catalyst-prod-45
cd catalyst-prod-45
npm install
2
Create .env.local
Copy staging env — already points to staging DB, no changes needed.
cp .env.staging .env.local
3
Start dev server
npm run dev   # → http://localhost:8080
Port is pinned to 8080. Never change it.
4
Get Supabase access Optional — only if doing DB/fn work
Ask Vikram to invite you to the staging Supabase project. Then:
brew install supabase/tap/supabase
supabase login

Branch model

Branch Purpose DB Frontend Deploy how
main Active development — push here Staging localhost:8080 npm run dev
feature/* Feature branches — open PR → merge to main Staging localhost:8080 PR review
production Frozen release — CI deploys to Vercel/ksa-catalyst.com Production ksa-catalyst.com Admin panel only
⚠ Never push directly to production branch. All production deploys go through /admin/connections/vercel → Promote branch. This runs the GitHub API merge, applies DB migrations, and triggers Vercel build automatically.

Rules — do not break these

RuleWhy
Never push to production branch CI runs DB migrations automatically on push. Wrong code = broken prod DB
Never commit .env.local or service role keys .env.local is gitignored. Service role key bypasses all RLS
Never change dev server port from 8080 Pinned in vite.config — changing breaks shared tooling
Never run migrations against prod without Vikram Production DB migrations cannot be auto-rolled back
