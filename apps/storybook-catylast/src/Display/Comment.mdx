<Subtitle>
  The conversation primitive across Catylast — issue threads, document
  comments, code-review notes, audit-log entries with prose. Pairs an
  avatar, an author name, an optional role tag, a timestamp, the comment
  body, an actions row, and an optional restriction marker, plus
  indented children for nested replies.
</Subtitle>

`Comment` ships from **`@catylast/primitives`**. It is built
around composition: every visual region is a slot accepting any React
node, so consumers can swap in router-aware author links, rich-text
bodies, custom badges, time-ago tooltips, or whatever the surrounding
surface needs. The component owns layout, typography, and the
conversational visual feel; the *content* of each slot is yours.

Every styling dimension is exposed as both an enum prop and a CSS
variable so consumers can override per-instance via `style` without
touching the prop API.

## Anatomy

```text
┌──────────────────────────────────────────────────────────────────┐
│  ┌────┐  Author Name  [Tag]  · 🔒 Restricted to X  · 2h ago      │  ← header
│  │ AV │                                                           │
│  └────┘  Content goes here. Plain text, rich text, anything.     │  ← body
│                                                                   │
│          Reply · React · Edit · Delete                            │  ← actions
│                                                                   │
│          ◆ Saving…                                                │  ← saving (optional)
│                                                                   │
│          ┌────────────────────────────────────────────────────┐   │
│          │  ┌────┐  Reply Author  · 1h ago                    │   │  ← nested reply
│          │  │ AV │  Reply content…                            │   │
│          │  └────┘  Reply · React                             │   │
│          └────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────────┘
```

| Slot              | Prop             | Purpose                                                                   |
| ----------------- | ---------------- | ------------------------------------------------------------------------- |
| **Avatar**        | `avatar`         | Visual identity. Pair with `<Avatar>` from primitives.                    |
| **Author**        | `author`         | Required. Author name; pass a node for a router-aware profile link.       |
| **Type**          | `type`           | Optional role tag — *Author*, *OP*, *Reviewer*, *Customer*.               |
| **Restricted**    | `restrictedTo`   | Lock icon + label when the comment is visibility-restricted.              |
| **Time**          | `time`           | Timestamp, typically a relative time linked to the permalink.             |
| **Content**       | `content`        | The comment body. Plain text, rich text, or any custom node.              |
| **Actions**       | `actions`        | Array of action items; rendered with `·` separators between siblings.     |
| **Saving**        | `isSaving`       | Animated dot + label under the body. Disables interaction.                |
| **Nested**        | `children`       | Replies indented in a thread block beneath the body.                      |

## Slots

### Avatar + time

Pair the comment with the design-system `<Avatar>` for visual identity,
and pass any node into `time` (typically a relative timestamp linking
to the permalink).

<Canvas of={CommentStories.WithAvatarAndTime} />

<Source
  language="tsx"
  code={`<Comment
  avatar={<Avatar name="Maya Patel" size="md" />}
  author="Maya Patel"
  time={<a href="/c/123">2 hours ago</a>}
  content="…"
  actions={[
    <Button appearance="subtle-link" size="small" spacing="none">Reply</Button>,
    <Button appearance="subtle-link" size="small" spacing="none">React</Button>,
  ]}
/>`}
/>

### Role tag

The `type` slot adds a small role tag right after the author name.
Useful for *Author*, *OP*, *Reviewer*, *Customer* — any context the
reader should see at a glance. Pass a string for the default tag style,
or any node (a `<Badge>`, a custom pill) for full control.

<Canvas of={CommentStories.WithRoleTag} />

<Source
  language="tsx"
  code={`<Comment author="Wasim Khan" type="Author" content="…" />

<Comment
  author="Sarah Lee"
  type={<Badge variant="primary">Reviewer</Badge>}
  content="…"
/>`}
/>

### Restricted

`restrictedTo` renders a lock icon and the supplied label, signalling
the comment is only visible to a specific audience. Use the warning
text color so the restriction reads as an advisory rather than an
error.

<Canvas of={CommentStories.Restricted} />

<Source
  language="tsx"
  code={`<Comment
  author="Sarah Lee"
  restrictedTo="Engineering only"
  content="Internal note — needs sign-off from infra before we merge."
/>`}
/>

### Actions

`actions` is an array of nodes rendered after the body, separated by
`·`. Buttons styled with `appearance="subtle-link"` blend in with the
prose; falsy entries (`null`, `undefined`, `false`) are filtered, so
conditional rendering with `cond && <Button />` Just Works.

<Source
  language="tsx"
  code={`<Comment
  author="Wasim"
  content="…"
  actions={[
    <Button appearance="subtle-link" size="small" spacing="none">Reply</Button>,
    canEdit && <Button appearance="subtle-link" size="small" spacing="none">Edit</Button>,
    canDelete && <Button appearance="subtle-link" size="small" spacing="none">Delete</Button>,
  ]}
/>`}
/>

## States

### Highlighted

Set `highlighted` to draw an accent border on the leading edge and
tint the background. Useful when the comment was *just posted*,
*mentions the current user*, or *is the focus of a deep link*.

<Canvas of={CommentStories.Highlighted} />

### Saving

While the comment is being persisted, set `isSaving`. The component
shows an animated dot + label under the body and disables interaction
so users can't double-submit. Override the copy via `savingText`.

<Canvas of={CommentStories.Saving} />

<Source
  language="tsx"
  code={`<Comment
  author="Priya Sharma"
  content="Updated the migration plan."
  isSaving
  savingText="Posting…"
/>`}
/>

## Nested replies

Compose `<Comment>` instances inside another `<Comment>` to render a
thread. The child block is indented and gets a hairline thread-guide
on its leading edge when you set `--comment-nested-border-color` /
`--comment-nested-border-width`.

<Canvas of={CommentStories.NestedReplies} />

<Source
  language="tsx"
  code={`<Comment author="Alex" content="Should we lift the card composition…?">
  <Comment author="Maya" content="Yes — but let's wait for the board view.">
    <Comment author="Wasim" type="Author" content="Agreed." />
  </Comment>
  <Comment author="Ben" content="One concern: backlog rows render hundreds…" />
</Comment>`}
/>

> **Tip:** keep nesting to **two or three levels max**. Deeper threads
> become hard to scan; flatten further replies to the same level with
> an "@user" mention prefix instead.

## Accessibility

- **`<article>`** root with the proper landmark semantics so screen
  readers can navigate between comments via the rotor / virtual
  cursor.
- **Author name as a heading** (`<h4>` by default) so the document
  outline reflects each comment as a navigable section. Override the
  level via `headingLevel` to fit the surrounding outline.
- **Restriction label** carries an `aria-label` like
  *"Restricted: Engineering only"* so the lock icon is announced.
- **Saving overlay** is `aria-live="polite"` so screen-reader users
  hear when the comment is persisting.
- **Action group** uses `role="group"` with `aria-label="Comment
  actions"` so it announces as a coherent set rather than a list of
  loose buttons.
- **No focus traps.** The Comment primitive owns layout only — every
  interactive element you slot in (links, buttons, menus) keeps its
  native focus behaviour.

## Best practices

- **Always include an avatar in conversational threads.** Even an
  initials fallback dramatically improves scannability when there are
  many participants.
- **Use `type` sparingly.** Reserve it for distinctions that change
  how a reader weighs the message (Author vs Reviewer vs Customer).
  Don't use it as a generic title.
- **Prefer relative timestamps in the body, absolute on hover.** Wrap
  the time slot in a `<Tooltip>` that surfaces the absolute timestamp
  for users who need precision.
- **Keep nesting to two or three levels.** Deeper threads become hard
  to scan; flatten further replies with an "@user" mention prefix.
- **Pair `restrictedTo` with backend enforcement.** The component
  signals visibility but does not enforce it — your API must filter
  restricted comments on the server.
- **Don't render dozens of `isSaving` comments at once.** The pulse
  animation is intentionally subtle; many running in parallel start to
  read as visual noise.

## Theme switching

Use the toolbar at the top of the canvas to flip between **Light**,
**Dark**, and **System**. The `data-theme` attribute on `<html>`
drives every token value the comment consumes.
