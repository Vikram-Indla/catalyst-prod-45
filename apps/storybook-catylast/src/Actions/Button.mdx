<Subtitle>
  The full button family for Catylast — Button, IconButton, LinkButton,
  LinkIconButton, SplitButton, and ButtonGroup. Eight appearances, three
  sizes, three spacing densities, full state model, polymorphic `as`,
  fully customisable through props *and* CSS variables.
</Subtitle>

`Button` is the workhorse interactive primitive across Catylast — every
Save, Cancel, Submit, "+ Create", filter, view-mode toggle, and inline
action flows through it. The family ships from
**`@catylast/primitives`**:

| Component         | Use it when                                                                   |
| ----------------- | ----------------------------------------------------------------------------- |
| `Button`          | Text + optional icon. The default for any clickable action.                   |
| `IconButton`      | Square, icon-only. Toolbars, dense surfaces, table row-actions.               |
| `LinkButton`      | Renders as `<a>` (or a router Link) but styled like a Button.                 |
| `LinkIconButton`  | IconButton rendered as an anchor.                                             |
| `SplitButton`     | Primary action + dropdown trigger sharing a vertical seam.                    |
| `ButtonGroup`     | Multiple buttons grouped as one widget — gapped or segmented, h or v.         |

Every appearance, size, spacing, state, and styling dimension is
**customisable through both an enum prop and a CSS variable**. So when
the design system's vocabulary doesn't include exactly what you need,
you can override any single knob without forking the component or
abandoning the design tokens.

## Quickstart

The minimum viable button — children only, no props:

<Canvas of={ButtonStories.Default} />

<Source
  language="tsx"
  code={`import { Button } from "@catylast/primitives";

<Button onClick={save}>Save changes</Button>;`}
/>

That renders the `default` appearance at `medium` size with `default`
spacing. From here you tune **appearance** (color preset),
**size** (overall scale), **spacing** (horizontal padding), or any of
the state flags.

---

## Appearances

Eight semantic color presets. Pick the one that matches the action's
intent — not its color.

<Canvas of={ButtonStories.Appearances} />

| Appearance      | When to use it                                                             |
| --------------- | -------------------------------------------------------------------------- |
| `default`       | The workhorse. 90% of buttons. Subtle filled surface with a border.        |
| `primary`       | The single most-important action on a surface (Save, Submit, Create).     |
| `subtle`        | Low-emphasis affordance inside a busy UI (Cancel, secondary toolbars).    |
| `subtle-link`   | Inline action that should look like text but behave like a button.        |
| `link`          | Inline link-styled button — accent color, underline on hover.             |
| `warning`       | Cautionary action that is reversible (Move to archive).                   |
| `danger`        | Destructive, irreversible action (Delete permanently).                    |
| `discovery`     | Special / AI / new-feature affordances. Purple. Use sparingly.            |

> **Rule of thumb:** at most one `primary` button per surface. If you
> have two equally-important actions, both are `default` — the visual
> hierarchy comes from order, not weight.

---

## Sizes

Three overall scales. Size affects font size, minimum height, and icon
glyph size — independent of `spacing` (which controls horizontal
padding).

<Canvas of={ButtonStories.Sizes} />

| Size     | Min height | When to reach for it                                          |
| -------- | ---------- | ------------------------------------------------------------- |
| `small`  | 24px       | Dense surfaces — table cells, toolbars, inline editors.      |
| `medium` | 32px       | Most contexts. Default.                                       |
| `large`  | 40px       | Hero CTAs, primary onboarding, marketing surfaces.            |

> Legacy `sm` / `md` / `lg` size values are still accepted; they map
> to `small` / `medium` / `large`. New code should use the long form.

---

## Spacing

Horizontal padding density, independent of size. Use to fine-tune
buttons inside dense surfaces without scaling the entire button down.

<Canvas of={ButtonStories.Spacing} />

| Spacing    | Effect                                                            |
| ---------- | ----------------------------------------------------------------- |
| `default`  | The size's standard padding (e.g. 12px on `medium`).              |
| `compact`  | Tightens horizontal padding to 6px.                               |
| `none`     | Zero padding — useful for `link` / `subtle-link` appearances.     |

## Icons

`iconBefore` and `iconAfter` accept either a registered icon name
(string) or any React node. Icon size auto-scales with button size.

<Canvas of={ButtonStories.Icons} />

<Source
  language="tsx"
  code={`<Button appearance="primary" iconBefore="check">
  Approve
</Button>

<Button appearance="default" iconAfter="chevron-down">
  More options
</Button>

<Button appearance="discovery" iconBefore={<Sparkles size={16} />}>
  Custom icon node
</Button>`}
/>

For icon-only buttons, reach for `<IconButton>` (next section) instead
of an empty-children `<Button>`.

## States

`isDisabled`, `isLoading`, `isSelected`, `shouldFitContainer` — all
opt-in.

<Canvas of={ButtonStories.States} />

- **isDisabled** — reduced opacity, no clicks, no focus on Tab. On
  native `<button>` we set the `disabled` attribute; on anchors and
  custom elements we set `aria-disabled="true"`.
- **isLoading** — spinner over the label. Width is preserved (no layout
  shift). Clicks suppressed during the loading state.
- **isSelected** — toggle-on. Accent border + tinted background. Pair
  with `aria-pressed` semantics for true toggles. The component sets
  `aria-pressed` automatically when `isSelected` is true.
- **shouldFitContainer** — stretches the button to fill its parent.
  Use for full-width form submit buttons on mobile or in narrow side
  panels.

<Canvas of={ButtonStories.FullWidth} />

> Legacy `disabled` (the native HTML attribute) is also supported for
> backwards compatibility. New code should use `isDisabled` for parity
> with the rest of the boolean state props.

## Polymorphic `as`

Render the button as a different element or component without losing
the button styling.

<Canvas of={ButtonStories.PolymorphicAs} />

<Source
  language="tsx"
  code={`// Native anchor
<Button as="a" href="/docs" appearance="primary">Read docs</Button>

// Router link (Next.js / React Router)
<Button as={NextLink} href="/docs" appearance="primary">Read docs</Button>

// Render as a non-interactive element (rare — make sure semantics fit)
<Button as="div" appearance="subtle">Visual button</Button>`}
/>

For the common case of a Button that navigates, prefer `LinkButton`
(below) — it pre-sets `as="a"` and types anchor props correctly.

## SplitButton

Two adjacent buttons sharing a vertical seam — a primary action on the
left and a dropdown trigger on the right. The component is a thin
container; it owns the seam treatment but **not** the menu — wrap a
real `<Menu>` around the right half so you get keyboard navigation,
focus trapping, and screen-reader semantics for free.

<Canvas of={ButtonStories.SplitButtons} />

<Source
  language="tsx"
  code={`import {
  SplitButton,
  Button,
  IconButton,
  Menu,
  MenuTrigger,
  MenuContent,
  MenuItem,
} from "@catylast/primitives";

<SplitButton label="Save options">
  <Button appearance="primary" onClick={save}>
    Save
  </Button>
  <Menu>
    <MenuTrigger asChild>
      <IconButton
        icon="chevron-down"
        label="More save options"
        appearance="primary"
      />
    </MenuTrigger>
    <MenuContent align="end">
      <MenuItem onSelect={saveAndContinue}>Save and continue</MenuItem>
      <MenuItem onSelect={saveAsDraft}>Save as draft</MenuItem>
      <MenuItem onSelect={saveAndClose}>Save and close</MenuItem>
    </MenuContent>
  </Menu>
</SplitButton>;`}
/>

> **Match both halves' `appearance`** for visual consistency. The
> component does not enforce this — exotic combinations (primary
> action + subtle chevron) remain available if you really want them.

## ButtonGroup

Multiple buttons grouped as one widget. Two modes:

### Gapped

Default — a horizontal (or vertical) row with consistent gap between
children. Use for related-but-distinct actions on a form footer.

<Canvas of={ButtonStories.ButtonGroupGapped} />

### Segmented

`isSegmented` collapses the buttons into a single segmented control —
adjacent borders share a seam, outside corners round as a unit, and
the focused / selected segment lifts above its neighbors. Use for
filters, view-modes, sort-orders that read as one widget.

<Canvas of={ButtonStories.ButtonGroupSegmented} />

### Vertical

`orientation="vertical"` stacks the buttons. Combine with `isSegmented`
for a side-rail switcher (think Figma's left rail).

<Canvas of={ButtonStories.ButtonGroupVertical} />

<Source
  language="tsx"
  code={`<ButtonGroup label="View mode" isSegmented>
  <Button appearance="default" isSelected={view === "list"}>List</Button>
  <Button appearance="default" isSelected={view === "board"}>Board</Button>
  <Button appearance="default" isSelected={view === "calendar"}>Calendar</Button>
</ButtonGroup>

<ButtonGroup label="Side rail" orientation="vertical" isSegmented>
  <IconButton icon="panel-left" label="Home" size="large" />
  <IconButton icon="search" label="Search" size="large" />
  <IconButton icon="settings" label="Settings" size="large" />
</ButtonGroup>`}
/>

## CSS variable escape hatch

Every prop maps to a CSS variable on the root element. To go beyond the
enum vocabulary — exact pixel sizes, brand colors, custom shadows —
override the same variables directly via `style`:

<Canvas of={ButtonStories.CssVariableEscapeHatch} />

<Source
  language="tsx"
  code={`<Button
  style={{
    "--btn-bg": "var(--catylast-color-purple-500)",
    "--btn-bg-hover": "var(--catylast-color-purple-600)",
    "--btn-color": "white",
    "--btn-radius": "999px",
    "--btn-padding-x": "20px",
    "--btn-shadow": "0 6px 18px rgba(120, 80, 200, 0.35)",
  }}
>
  Branded pill
</Button>`}
/>

### Variables published

| Variable                    | Default                | Purpose                                     |
| --------------------------- | ---------------------- | ------------------------------------------- |
| `--btn-bg`                  | per appearance         | Background color.                           |
| `--btn-bg-hover`            | per appearance         | Hover background.                           |
| `--btn-bg-pressed`          | per appearance         | Active / pressed background.                |
| `--btn-bg-selected`         | `surface.selected`     | `isSelected` background override.           |
| `--btn-color`               | per appearance         | Text + icon color.                          |
| `--btn-color-hover`         | inherit                | Text color on hover.                        |
| `--btn-color-selected`      | `text.accent`          | `isSelected` text color override.           |
| `--btn-border-color`        | per appearance         | Border color.                               |
| `--btn-border-color-hover`  | per appearance         | Border color on hover.                      |
| `--btn-border-color-selected` | `border.focus`       | `isSelected` border color.                  |
| `--btn-border-width`        | `1px`                  | Border width.                               |
| `--btn-radius`              | `radius.sm`            | Corner radius.                              |
| `--btn-shadow`              | `none`                 | Shadow.                                     |
| `--btn-padding-x`           | per size               | Horizontal padding.                         |
| `--btn-padding-y`           | per size               | Vertical padding.                           |
| `--btn-min-height`          | per size               | Minimum height.                             |
| `--btn-width`               | `auto` / `100%`        | `shouldFitContainer` toggle.                |
| `--btn-font-size`           | per size               | Label font size.                            |
| `--btn-font-weight`         | `medium`               | Label font weight.                          |
| `--btn-gap`                 | per size               | Gap between icon and label.                 |
| `--btn-icon-size`           | per size               | Icon glyph size.                            |
| `--btn-text-decoration`     | `none`                 | Text decoration (used by link variants).    |
| `--btn-text-decoration-hover` | per appearance       | Text decoration on hover.                   |

For `IconButton`, the same `--btn-*` variables apply, plus `--btn-size`
which controls the square dimensions.

For `SplitButton`, `--split-radius` and `--split-divider-color` control
the seam treatment.

For `ButtonGroup`, `--btn-group-radius` and `--btn-group-divider-color`
control the segmented mode's outer radius and inner seam color.

## Accessibility

- **Native `<button>` semantics by default.** All button family
  components render as `<button type="button">` unless `as` is
  overridden.
- **Anchors get `aria-disabled`** instead of the (non-existent on `<a>`)
  `disabled` attribute when `isDisabled` is set.
- **Selection** announces via `aria-pressed="true"` on the button.
- **Loading buttons** suppress click handling and set
  `data-loading="true"` for styling. The native click handler is fully
  blocked — you don't need to debounce client-side.
- **IconButton requires a `label` prop** which becomes `aria-label` so
  screen-reader users always know what the button does.
- **ButtonGroup** has `role="group"` and `aria-label`. Adjacent
  segments lift out via `z-index` on focus / selection so the focus
  ring isn't clipped by neighboring borders.
- **SplitButton** is `role="group"` with `aria-label`. The two halves
  remain independent focus targets — the menu trigger behaves exactly
  like any other `<MenuTrigger>` so keyboard users get the standard
  arrow-key navigation when the menu opens.
- **Focus rings** are token-driven (`border.focus`) and survive theme
  flips automatically.

## Best practices

- **One `primary` per surface.** The visual hierarchy collapses if
  every button is primary.
- **Lead with text.** Don't replace a button label with just an icon
  unless space is genuinely scarce — and when you do, reach for
  `<IconButton>` so the `label` prop forces an `aria-label`.
- **Pick `appearance` from intent, not color.** "Confirm action that
  will delete data" is `danger`, regardless of the surrounding palette.
- **Pair `Button` with the same appearance inside `SplitButton` and
  `ButtonGroup`.** Mixed appearances inside a group rarely read as
  cohesive.
- **Use `isLoading` for actions that round-trip a network.** It's
  cheaper than a separate spinner overlay, blocks duplicate clicks,
  and preserves the button's width so the surrounding layout doesn't
  jump.
- **Don't fake disabled with CSS.** If a button shouldn't be clicked,
  set `isDisabled` so the click handler is suppressed and the button
  is removed from the tab order. Users should never see a disabled-
  looking button that still fires its action.

## Migration from v0.1

The v0.1 API used `variant` and short-form sizes. Both still work — the
new API just gives you more knobs.

| v0.1 prop        | v0.2 prop                                                |
| ---------------- | -------------------------------------------------------- |
| `variant="primary"`   | `appearance="primary"`                              |
| `variant="secondary"` | `appearance="default"` (or keep `variant="secondary"`) |
| `variant="ghost"`     | `appearance="subtle"` (or keep `variant="ghost"`)   |
| `variant="danger"`    | `appearance="danger"`                               |
| `size="sm"` / `"md"` / `"lg"` | `size="small"` / `"medium"` / `"large"`     |
| `disabled`             | `isDisabled` (both still work)                     |

No callsites need to change. Eventually, drop `variant` and the short
size aliases when comfortable.

## Theme switching

Use the toolbar at the top of the canvas to flip between **Light**,
**Dark**, and **System**. The `data-theme` attribute on `<html>` drives
every token value the buttons consume; nothing in this component reads
the theme name directly.

## Related

- **`@catylast/tokens`** — every default value flows from here.
  Override the underlying token to retune every button on the page at
  once.
- **`@catylast/icons`** — icon glyphs accepted by `iconBefore` /
  `iconAfter` / `IconButton.icon` as string names.
- **`Menu`** — pair with `SplitButton` (and any `Button` /
  `IconButton`) to build dropdowns, action menus, and toolbars.
- **`Tooltip`** — wrap an `IconButton` with `Tooltip` to surface its
  `label` on hover for sighted users too.
