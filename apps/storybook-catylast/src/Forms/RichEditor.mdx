<Subtitle>
  A Tiptap-backed rich text editor with a click-to-edit pattern,
  dynamic toolbar (preset / custom array / hidden), slash menu, mentions,
  panels, callouts, and a custom-built block drag-and-drop layer.
</Subtitle>

`RichEditor` is the long-form text input across Catylast — issue
descriptions, comment threads, document pages, release notes, project
briefs. It renders **read-only by default**; clicking the content switches
to edit mode (toolbar appears, content becomes editable, **Save** /
**Cancel** buttons render at the bottom).

The editor is built on **Tiptap v3** (ProseMirror) for the headless engine,
but every visual layer — toolbar, drag handle, slash menu, insert-element
modal, emoji picker, link bubble menu — is owned by Catylast. Tokens-driven
styling means dark mode and theme swaps work without component changes.

<Source

## Quickstart

The minimum viable editor — `defaultValue` + `onSave`:

<Canvas of={RichEditorStories.Default} />

<Source
  language="tsx"
  code={`import { RichEditor } from "@catylast/rich-editor";

<RichEditor
  defaultValue="<p>Hello, Catylast.</p>"
  onSave={async ({ html, json }) => {
    await api.saveDescription({ html, json });
  }}
/>;`}
/>

Click the content to enter edit mode. Save commits via `onSave`
(receives both HTML and Tiptap JSON); Cancel reverts to the last saved
state.

## Slash menu

Type **`/`** anywhere in the editor to open the slash menu. A searchable
list of insertable blocks appears (headings, lists, code, blockquote,
horizontal rule, panels). Use **arrow keys** + **Enter** to pick, or
click. The default command set covers everything in the toolbar; consumers
can override with the `slashCommands` prop.

<Canvas of={RichEditorStories.SlashMenu} />

<Source
  language="tsx"
  code={`<RichEditor
  slashCommands={[
    {
      id: "heading-1",
      label: "Heading 1",
      keywords: ["h1", "title"],
      run: ({ editor }) =>
        editor.chain().focus().toggleHeading({ level: 1 }).run(),
    },
    // …more commands
  ]}
/>`}
/>

Pass `slashCommands={false}` to disable the slash menu entirely.

## Mentions

Type **`@`** to open the mention popover. The list filters as you type by
name. Pick with arrow keys + Enter. Each mention renders as a styled pill
in the document. The user list comes from your
`getMentionSuggestions(query)` callback — return synchronously or as a
Promise.

<Canvas of={RichEditorStories.Mentions} />

<Source
  language="tsx"
  code={`<RichEditor
  getMentionSuggestions={async (query) => {
    const users = await api.searchUsers(query);
    return users.map((u) => ({
      id: u.id,
      label: u.fullName,
      description: u.team,
    }));
  }}
/>`}
/>

If `getMentionSuggestions` is omitted, the mention extension is not
loaded — the `@` toolbar button is disabled.

## Media + attachments upload

The editor speaks a single uploader API for **images, videos, and any
other file type**. The `onUpload(file)` callback runs on every upload
— from the **image** / **video** / **paperclip** toolbar buttons, the
**File attachment** entry in the +Insert menu, and (in a future
iteration) drag-and-drop. Your handler decides what kind of node to
insert by setting the `type` field on the result.

| `type`     | Inserts                                                                |
| ---------- | ---------------------------------------------------------------------- |
| `"image"`  | inline `<img>` (Tiptap Image extension).                               |
| `"video"`  | inline `<video controls>` (Catylast Video extension).                  |
| `"file"`   | a Catylast Attachment card — colored extension tile + filename + size + download. |

The `name`, `size`, and `mimeType` fields on `UploadResult` are
optional — if present, they drive the Attachment card's filename, the
human-readable size label ("1.4 MB"), and the icon-tile color and text.

<Canvas of={RichEditorStories.MediaUpload} />

<Source
  language="tsx"
  code={`<RichEditor
  onUpload={async (file) => {
    const url = await api.uploadFile(file);
    if (file.type.startsWith("image/")) {
      return { url, type: "image", name: file.name, size: file.size, mimeType: file.type };
    }
    if (file.type.startsWith("video/")) {
      return { url, type: "video", name: file.name, size: file.size, mimeType: file.type };
    }
    return {
      url,
      type: "file",
      name: file.name,
      size: file.size,
      mimeType: file.type,
    };
  }}
/>`}
/>

If `onUpload` is omitted, the toolbar falls back to a simple URL prompt
for each kind — the user types in a URL and the editor inserts the
appropriate node.

## File attachments

The Attachment node renders a card-like row with a colored extension
tile, the filename, a meta line ("1.4 MB · PDF document"), and a
trailing download button. The same Attachment block works for any file
type — PDFs, ZIPs, docs, spreadsheets, source code, audio, anything
your `onUpload` returns with `type: "file"`.

<Canvas of={RichEditorStories.Attachments} />

The colored tile is keyed off the file kind (chosen from `mimeType`
and the filename extension):

| Kind           | Tile                                      | Examples                                           |
| -------------- | ----------------------------------------- | -------------------------------------------------- |
| `image`        | purple                                    | PNG, JPG, GIF, WebP, SVG                           |
| `video`        | blue                                      | MP4, WebM, MOV                                     |
| `audio`        | green                                     | MP3, WAV, OGG, M4A                                 |
| `pdf`          | red                                       | PDF                                                |
| `archive`      | yellow                                    | ZIP, RAR, TAR, GZ, 7Z                              |
| `document`     | blue                                      | DOC, DOCX, RTF, ODT, TXT, MD                       |
| `spreadsheet`  | green                                     | XLS, XLSX, CSV, ODS                                |
| `presentation` | dark yellow                               | PPT, PPTX, ODP, KEY                                |
| `code`         | charcoal                                  | JS / TS / PY / JSON / YAML / HTML / CSS / …        |
| `file`         | neutral (default fallback)                | anything else                                      |

The kind classification is exposed as the `pickKind(mimeType, name)`
helper from `@catylast/rich-editor` so consumers can implement custom
upload UIs that match the editor's visual language.

### Inserting attachments programmatically

The `Attachment` extension exposes a `setAttachment` Tiptap command so
consumers can drop attachment nodes from outside the toolbar (e.g. a
custom drop zone, a paste handler, a "recent files" picker):

<Source
  language="tsx"
  code={`editor
  .chain()
  .focus()
  .setAttachment({
    url: "/uploads/release-notes.pdf",
    name: "release-notes.pdf",
    size: 284715,
    mimeType: "application/pdf",
  })
  .run();`}
/>

### Round-tripping

Attachments survive `getJSON()` / `getHTML()` round-trips losslessly —
the URL, name, size, and MIME type are all stored as node attributes
on the document. Saved HTML can be re-loaded into the editor or
rendered standalone (the `<a>` tag and the visual classes are part of
the markup the extension emits).

## Block drag-and-drop

Hover any top-level block in edit mode and a vertical grip appears in the
left gutter. Click and drag to move the block to a new sibling position —
drop above or below another block. The drop indicator (a thin accent line)
shows where the block will land.

<Canvas of={RichEditorStories.DragHandle} />

> **Built from scratch.** The drag handle is a custom Tiptap extension —
> no community drag-handle dependency. Source lives in
> `packages/rich-editor/src/extensions/DragHandle.ts`. This was a
> deliberate design decision so we could match the visual feel of the rest
> of the design system pixel-perfectly, without inheriting upstream
> styling conventions or branding.

Disable with `disableDragHandle`.

## Inline-edit pattern

The full click-to-edit flow end-to-end: read-only on mount → click →
toolbar + Save / Cancel appear → edit → Save commits via `onSave`.

<Canvas of={RichEditorStories.InlineEditPattern} />

<Source
  language="tsx"
  code={`function EditableDescription({ initialHtml }) {
  const [savedHtml, setSavedHtml] = useState(initialHtml);
  return (
    <RichEditor
      defaultValue={savedHtml}
      onSave={async ({ html }) => {
        await api.updateDescription(html);
        setSavedHtml(html);
      }}
    />
  );
}`}
/>

Use `saveLabel` / `cancelLabel` to override the button text. If `onSave`
is omitted, the Save / Cancel bar is not rendered (the editor still
toggles between view and edit, but commits are handled externally).

## Insert element menu

The toolbar's **+** button opens a compact dropdown of insertable elements
(icon + title + one-liner) with a search field at the top. Click **Browse
all** to open the full modal — sidebar with **All** / **Content** /
**Workspace** / **External** / **Development** categories, search, and a
card grid.

<Canvas of={RichEditorStories.InsertElementMenu} />

### Customize the menu

Pass `insertElements` to extend the menu with consumer-specific items. By
default they're appended to the built-in set; set `replaceInsertElements`
to swap the list entirely.

<Canvas of={RichEditorStories.CustomInsertElements} />

<Source
  language="tsx"
  code={`<RichEditor
  insertElements={[
    {
      id: "deploy-status",
      label: "Deploy status",
      description: "Live status badge for the latest deploy",
      icon: "rocket",
      iconTint: "var(--catylast-color-purple-100)",
      category: "development",
      keywords: ["release", "ship", "ci"],
      run: ({ editor }) =>
        editor.chain().focus().insertContent("[deploy]").run(),
    },
  ]}
/>`}
/>

Each item supplies an `icon` (`@catylast/icons` glyph name), `label`,
one-line `description`, `category` (`content` / `workspace` / `external`
/ `development`), optional `keywords`, optional `iconTint` (CSS color for
the icon tile in the modal grid), and a `run({ editor })` callback that
performs the insert.

## Save payload

`onSave` receives both serializations:

<Source
  language="tsx"
  code={`type SavePayload = {
  /** Tiptap's native JSON document. Best fidelity for round-tripping. */
  json: object;
  /** Rendered HTML string. Use for read-only display when JSON isn't available. */
  html: string;
};

<RichEditor
  onSave={async ({ json, html }) => {
    // Persist JSON for full fidelity (mentions, panels, etc. round-trip cleanly)
    await api.saveDocument({ json });
    // OR persist HTML for simple display surfaces (lists, search results)
    await api.saveDocument({ html });
  }}
/>`}
/>

> **Recommendation:** save JSON for the source of truth, render HTML for
> read-only preview surfaces (search results, list rows, notifications).
> JSON round-trips cleanly through the editor; HTML can lose extension-
> specific attributes (mentions, panels) if your sanitizer is too strict.

## Accessibility

- **Keyboard** — every toolbar button is reachable via Tab and triggered
  by Enter / Space. Common formatting shortcuts (Cmd/Ctrl+B, +I, +U, +K
  for link) work in edit mode.
- **Slash menu** — opened by `/`, closed by `Esc`. Arrow-up / Arrow-down
  navigate; Enter inserts; the highlighted item is announced via
  `aria-activedescendant`.
- **Mention popover** — same keyboard model as the slash menu. Each
  candidate is a real `<button>` so screen readers announce it.
- **Drag handle** — purely a pointer affordance today (v0.1). Keyboard
  reordering of blocks is on the roadmap.
- **Save / Cancel buttons** — labeled in plain English. The Save button
  shows a "Saving…" state and disables itself during the awaited
  `onSave` call.
- **Theme-agnostic colors** — every visual layer reads from
  `@catylast/tokens` semantic tokens, so light, dark, and high-contrast
  modes work without component changes.

## Best practices

- **Persist JSON, render HTML** — see the save-payload note above.
- **Pick the smallest toolbar that does the job** — `basic` for comments,
  `standard` for descriptions, `full` for documents. Custom arrays for
  surfaces that need a unique mix.
- **Always handle the loading state in `onSave`** — `RichEditor` shows a
  "Saving…" label automatically while the awaited `onSave` is in flight,
  but your network errors should be surfaced via toast or inline message.
- **Keep `getMentionSuggestions` fast** — debounce on the consumer side
  if your search hits the network, and return at most ~10 results.
- **Don't fight the slash menu** — if you've added consumer-specific
  insertables via `insertElements`, also surface them in `slashCommands`
  so users get to them via either path.
- **Use `readOnly` on display surfaces** — when content is shown but not
  edited (e.g. list rows, archived items), pass `readOnly` rather than
  hiding the toolbar — it's clearer to the reader that they can't edit.

## Theme switching

Use the toolbar at the top of the canvas to flip between **Light**,
**Dark**, and **System**. The `data-theme` attribute on `<html>` drives
every token value the editor consumes; nothing in this component reads the
theme name directly.
