<Subtitle>
  The date-grid primitive Catylast uses to build date pickers, range
  pickers, scheduling surfaces, and any feature that needs a month view.
  Single-select on the surface, but exposes a `selected[]` array so
  consumers can compose multi-day highlights, ranges, and "previously
  selected" trails on top of it.
</Subtitle>

`Calendar` ships from **`@catylast/primitives`**. It is built from
scratch on `Intl.DateTimeFormat` for locale-aware month and weekday
labels — **no `date-fns` / `moment` / `luxon` dependency**. The
component deals in *calendar* days (year/month/day triples) so DST
shifts and locale-specific midnight quirks don't affect picking.

Every styling dimension is exposed as both an enum prop and a CSS
variable so consumers can override per-instance via `style` without
touching the prop API. Defaults flow through `@catylast/tokens`
semantic tokens — dark mode and theme swaps work without component
changes.

## Quickstart

The minimum viable calendar — uncontrolled, opens on today's month:

<Canvas of={CalendarStories.Default} />

<Source
  language="tsx"
  code={`import { Calendar } from "@catylast/primitives";

<Calendar onSelect={(e) => console.log(e.iso)} />;`}
/>

The `onSelect` callback receives `{ iso, year, month, day }` — `iso` is
a `YYYY-MM-DD` string, `month` is zero-indexed (Jan = 0).

---

## Controlled vs uncontrolled

The visible month and focused date can be either uncontrolled (the
component owns the state) or fully controlled (you do). Pass either
the `default*` props or the `month`/`year`/`day` props — not both.

<Canvas of={CalendarStories.Controlled} />

<Source
  language="tsx"
  code={`// Uncontrolled — pick an initial month, the rest is internal state
<Calendar
  defaultMonth={4}      // May (zero-indexed)
  defaultYear={2026}
  defaultDay={12}
  onSelect={(e) => save(e.iso)}
/>

// Controlled — you own the visible month and the focused day
<Calendar
  month={state.month}
  year={state.year}
  day={state.day}
  onSelect={handlePick}
  onChange={(e) => {
    if (e.type === "navigate") setMonth({ year: e.year, month: e.month });
  }}
/>`}
/>

> **Tip:** `onChange` fires on every focus / navigation change (arrow
> keys, prev / next month). `onSelect` fires only when the user
> commits a pick (click, Enter, Space). Use `onChange` to follow the
> visible month; use `onSelect` to commit business state.

## Selection

`selected` is an array of ISO date strings to highlight. Use a single
value for date-picker behavior, or multiple values to surface a non-
contiguous selection set ("days you have meetings", "days the report
is generated", …).

<Canvas of={CalendarStories.Selected} />

<Source
  language="tsx"
  code={`// Single date — date-picker shape
<Calendar selected={["2026-05-12"]} onSelect={(e) => set(e.iso)} />

// Multiple dates — visual highlight set
<Calendar
  selected={["2026-05-04", "2026-05-08", "2026-05-15", "2026-05-22"]}
/>`}
/>

## Disabled dates

Pass `disabled` (an array of ISO date strings) to make specific dates
unselectable. Disabled dates show with reduced opacity and a
strikethrough, and don't fire `onSelect`.

## Min / max range

`minDate` and `maxDate` clamp the entire selectable range. Days
outside the range are disabled, and the prev / next month buttons
disable themselves when navigation would leave the range.

## Locale support

Month and weekday labels go through `Intl.DateTimeFormat`. Any BCP-47
locale tag works without an extra dependency.

> **RTL note:** the calendar reads correctly under `dir="rtl"` because
> the day grid uses logical CSS properties. Wrap the Calendar in a
> `dir="rtl"` parent for Arabic / Hebrew layouts.

## Keyboard model

The calendar implements the W3C grid pattern.

| Key                          | Action                          |
| ---------------------------- | ------------------------------- |
| **← / →**                    | Previous / next day             |
| **↑ / ↓**                    | Previous / next week            |
| **PageUp / PageDown**        | Previous / next month           |
| **Shift + PageUp / PageDown** | Previous / next year            |
| **Home / End**               | Start / end of the focused week |
| **Enter / Space**            | Select the focused date         |
| **Esc**                      | Blur the calendar               |

Tab steps in and out of the grid in one stop — the focused day cell
holds `tabindex="0"`; all other cells are `-1`. Within the grid the
arrow keys move the *roving* tabindex.

## Accessibility

- **`role="grid"`** on the root with the visible month as `aria-label`.
  Each week is a `role="row"`, each day a `role="gridcell"`.
- **`aria-current="date"`** marks today.
- **`aria-selected="true"`** marks selected days.
- **`aria-disabled="true"`** is set on disabled / out-of-range cells.
- **Long `aria-label`** on each cell gives the full date in the active
  locale (e.g. "Tuesday, 12 May 2026") so screen-reader users hear the
  day-of-week even when the grid only shows numbers.
- **Focus management:** the focused cell stays `tabindex="0"`. After
  keyboard navigation the focused button is programmatically focused
  so VoiceOver / NVDA / JAWS announce the new date.

## Recipes

### Date picker

Pair a `Calendar` with a trigger button. Catylast ships a dedicated
`DatePicker` on top of this primitive; the pattern is:

<Canvas of={CalendarStories.DatePickerRecipe} />

<Source
  language="tsx"
  code={`function DatePicker({ value, onChange }) {
  const [open, setOpen] = useState(false);
  return (
    <Popover open={open} onOpenChange={setOpen}>
      <PopoverTrigger asChild>
        <Button iconAfter="chevron-down">{value ?? "Pick a date"}</Button>
      </PopoverTrigger>
      <PopoverContent>
        <Calendar
          selected={value ? [value] : []}
          onSelect={(e) => {
            onChange(e.iso);
            setOpen(false);
          }}
        />
      </PopoverContent>
    </Popover>
  );
}`}
/>

### Range picker

Wrap the calendar in a tiny state machine that tracks `start` / `end`.
The first click sets `start` (and clears `end`); the second click sets
`end` and fills the days between as `selected`. The third click resets
and starts a new range.

<Canvas of={CalendarStories.RangeRecipe} />

<Source
  language="tsx"
  code={`const [start, setStart] = useState(null);
const [end, setEnd] = useState(null);

const range = useMemo(() => {
  if (!start) return [];
  if (!end) return [start];
  return enumerateIso(start, end);
}, [start, end]);

const onPick = (iso) => {
  if (!start || (start && end)) {
    setStart(iso); setEnd(null);
  } else if (iso < start) {
    setEnd(start); setStart(iso);
  } else {
    setEnd(iso);
  }
};

<Calendar selected={range} onSelect={(e) => onPick(e.iso)} />`}
/>

### Disable weekends

<Source
  language="tsx"
  code={`function weekendsForMonth(year, month) {
  const out = [];
  const last = new Date(year, month + 1, 0).getDate();
  for (let d = 1; d <= last; d++) {
    const wd = new Date(year, month, d).getDay();
    if (wd === 0 || wd === 6) {
      out.push(\`\${year}-\${String(month + 1).padStart(2, "0")}-\${String(d).padStart(2, "0")}\`);
    }
  }
  return out;
}

<Calendar disabled={weekendsForMonth(2026, 4)} />`}
/>

## Best practices

- **Pick the smallest size that fits.** A `large` calendar inside a
  popover wastes space and slows the user down.
- **Always set `onSelect` and `onChange` on controlled calendars.**
  `onSelect` for the user's pick, `onChange` to keep the visible month
  in sync as they navigate.
- **Use `previouslySelected`, not `disabled`, for "already picked"
  dates.** Disabled means *you can't pick this*; previously-selected
  means *you have picked this before, but it's still pickable*.
- **Always provide `minDate` / `maxDate` for booking flows.**
  Otherwise users can navigate to year 5000 or year 0 by mashing
  PageDown/Up.
- **Don't fight the locale.** If your app already has a locale
  setting (i18n), feed it directly to `Calendar` rather than forcing
  US English on every user.
