---
name: build-feature-packet
description: Fragment approved requirements into a bounded, independently testable Codex packet.
---

A valid packet has:
- one coherent user outcome;
- linked decision and requirement IDs;
- explicit in/out scope;
- discovery evidence;
- acceptance criteria;
- regression surfaces;
- security, accessibility, i18n, and observability considerations;
- test plan;
- rollback/recovery note;
- approval metadata.
